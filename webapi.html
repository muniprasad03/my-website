Top 100 .NET Core Web API Interview Questions

1. What is .NET Core?

.NET Core is a cross-platform, open-source framework developed by Microsoft for building modern applications. It supports multiple operating systems, including Windows, Linux, and macOS. .NET Core provides a modular and high-performance runtime and library for building various types of applications, including web APIs.

2. What is a Web API?

A Web API (Application Programming Interface) is a set of HTTP endpoints that allow applications to communicate with each other over the web. It provides a way for different systems to interact using standard web protocols, such as HTTP, and typically returns data in formats like JSON or XML.

3. What is the difference between ASP.NET Core and .NET Core?

.NET Core is a cross-platform runtime for building applications, while ASP.NET Core is a framework built on top of .NET Core specifically for creating web applications and APIs. ASP.NET Core leverages .NET Core for runtime services and libraries but includes additional features and libraries for web development.

4. How do you create a Web API in .NET Core?

To create a Web API in .NET Core, you can follow these steps:

  1. Create a new ASP.NET Core project using Visual Studio or the .NET CLI.
  2. Define your API controllers by creating classes that inherit from ControllerBase and use the [ApiController] attribute.
  3. Define your API methods using HTTP method attributes like [HttpGet], [HttpPost], etc.
  4. Configure routing and services in the Startup.cs file or Program.cs file (in .NET 6 and later).

5. What is dependency injection in .NET Core?

Dependency injection is a design pattern used to achieve Inversion of Control (IoC) between classes and their dependencies. In .NET Core, it is built-in and configured through the Startup.cs file. Services can be registered with different lifetimes (Transient, Scoped, Singleton) and then injected into constructors of classes that require them.

6. How does routing work in ASP.NET Core Web API?

Routing in ASP.NET Core Web API determines how HTTP requests are matched to action methods in controllers. Routing is configured in the Startup.cs file or Program.cs file using the UseRouting and UseEndpoints methods. Routes can be defined using attributes or convention-based routing.

7. What is middleware in ASP.NET Core?

Middleware in ASP.NET Core is software that is assembled into an application pipeline to handle requests and responses. Each component (middleware) in the pipeline has access to the HTTP context and can perform operations before and after the next component in the pipeline is invoked. Examples include authentication, logging, and exception handling middleware.

8. What are action results in ASP.NET Core Web API?

Action results in ASP.NET Core Web API represent the result of an action method. They are returned by the action methods and can be of various types such as OkResult, NotFoundResult, BadRequestResult, or ObjectResult. These results help in sending different HTTP status codes and response bodies back to the client.

9. How do you handle exceptions in ASP.NET Core Web API?

Exceptions in ASP.NET Core Web API can be handled using exception middleware. The UseExceptionHandler middleware can be configured in the Startup.cs file to catch exceptions globally and return a custom error response. Additionally, custom exception filters and try-catch blocks can be used for finer control over exception handling.

10. What is model binding in ASP.NET Core Web API?

Model binding in ASP.NET Core Web API is the process of converting HTTP request data (such as query strings, form data, or route data) into .NET objects. The framework binds the data from the request to action method parameters or model properties. This allows for easy and automatic data manipulation and validation in the application.

11. What is model validation in ASP.NET Core Web API?

Model validation in ASP.NET Core Web API ensures that the data sent to the API meets certain criteria before it is processed. Validation attributes, such as [Required], [StringLength], and [Range], can be applied to model properties. Validation results can be accessed via the ModelState object in controllers.

12. How do you enable Swagger in ASP.NET Core Web API?

Swagger can be enabled in an ASP.NET Core Web API project by adding the Swashbuckle.AspNetCore package to the project. In the Startup.cs file, call the services.AddSwaggerGen() method in ConfigureServices and app.UseSwagger() and app.UseSwaggerUI() in Configure to set up Swagger middleware.

13. What is the purpose of the `appsettings.json` file in .NET Core?

The `appsettings.json` file is used for storing configuration settings in a .NET Core application. It allows for hierarchical configuration of application settings and is read at runtime to provide configuration values to the application. It supports various formats like JSON and can be used to configure different aspects of the application, such as database connections and logging settings.

14. How do you perform unit testing in .NET Core?

Unit testing in .NET Core can be performed using frameworks like xUnit, NUnit, or MSTest. You can create test projects in Visual Studio or using the .NET CLI. Tests are written to verify the behavior of individual units of code (like methods or classes) in isolation from external dependencies. Dependency injection and mocking frameworks, such as Moq, can be used to help with testing.

15. What is the difference between `IActionResult` and `ActionResult` in ASP.NET Core Web API?

IActionResult is an interface that defines the contract for action results in ASP.NET Core, while ActionResult is a concrete class that implements IActionResult. Using IActionResult allows for more flexible and testable code, whereas ActionResult provides a richer set of methods and properties for action results.

87. What is the difference between `Select` and `SelectMany` in LINQ?

The Select method in LINQ projects each element of a sequence into a new form. It returns a sequence where each element is transformed based on the provided selector function. The SelectMany method, on the other hand, projects each element of a sequence to an IEnumerable and flattens the resulting sequences into a single sequence. SelectMany is used for working with nested collections or sequences.

99. How do you perform a left join in LINQ?

Performing a left join in LINQ can be done using the DefaultIfEmpty method in combination with GroupJoin: 

var leftJoin = from o in orders
                                  join c in customers on o.CustomerId equals c.Id into gj
                                  from subc in gj.DefaultIfEmpty()
                                  select new { o.OrderId, CustomerName = subc?.Name };

This will return all orders, including those that do not have a matching customer.
