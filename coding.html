C# OOP Coding Interview Questions

1. What is Object-Oriented Programming (OOP)?

Object-Oriented Programming (OOP) is a programming paradigm based on the concept of "objects" that contain data and methods. It promotes principles such as encapsulation, inheritance, polymorphism, and abstraction.

2. Explain the four fundamental OOP concepts.

  • Encapsulation: Hiding the internal state and requiring all interaction to be performed through an object's methods.
  • Inheritance: Allowing a new class to take on the properties and methods of an existing class.
  • Polymorphism: Enabling a single action to behave differently based on the object that it is acting upon.
  • Abstraction: Simplifying complex systems by modeling classes appropriate to the problem.

3. How do you implement inheritance in C#?

Inheritance in C# is implemented using the colon (:) symbol. The derived class inherits the members of the base class.

class Animal { public void Eat() { Console.WriteLine("Eating"); } } class Dog : Animal { public void Bark() { Console.WriteLine("Barking"); } }

4. What is an interface in C# and how do you use it?

An interface in C# is a contract that defines a set of methods and properties that the implementing class must provide. It is defined using the interface keyword.

interface IAnimal { void Eat(); } class Dog : IAnimal { public void Eat() { Console.WriteLine("Eating"); } }

5. Explain polymorphism with an example.

Polymorphism allows methods to do different things based on the object it is acting upon, even though they share the same name.

class Animal { public virtual void MakeSound() { Console.WriteLine("Some sound"); } } class Dog : Animal { public override void MakeSound() { Console.WriteLine("Bark"); } } class Cat : Animal { public override void MakeSound() { Console.WriteLine("Meow"); } } // Usage Animal myDog = new Dog(); Animal myCat = new Cat(); myDog.MakeSound(); // Outputs: Bark myCat.MakeSound(); // Outputs: Meow

6. What is the difference between an abstract class and an interface?

  • Abstract Class: Can have both abstract and concrete methods, can contain fields and constructors, supports access modifiers.
  • Interface: Can only have abstract methods (until C# 8.0), cannot contain fields or constructors, and does not support access modifiers (all members are public by default).

7. How does encapsulation improve the robustness of your code?

Encapsulation improves code robustness by hiding the internal state of an object and only allowing modification through well-defined methods. This ensures that the object maintains a valid state and prevents unauthorized access and modifications.

8. Can you provide an example of using encapsulation in C#?

class Person { private string name; private int age; public string Name { get { return name; } set { name = value; } } public int Age { get { return age; } set { if (value >= 0) age = value; } } } // Usage Person person = new Person(); person.Name = "John"; person.Age = 30;

9. What is a constructor and what is its purpose in C#?

A constructor is a special method in a class that is called when an instance of the class is created. Its purpose is to initialize the object.

class Person { public string Name { get; private set; } public Person(string name) { Name = name; } }

10. What are static classes in C#? Can we create objects of a static class?

Static classes are classes that cannot be instantiated and are used to create utility or helper classes that contain static members. No, you cannot create objects of a static class.

public static class Utility { public static void DoWork() { Console.WriteLine("Doing work."); } } // Usage Utility.DoWork(); // Correct // Utility util = new Utility(); // Incorrect - This will cause a compilation error.

11. What is a sealed class in C#?

A sealed class is a class that cannot be inherited. It is used to prevent further derivation of a class, often for security or design reasons.

sealed class SealedClass { public void Display() { Console.WriteLine("Sealed class method"); } }

12. What is the difference between a field and a property in C#?

  • Field: A variable that is declared directly in a class.
  • Property: A member that provides a flexible mechanism to read, write, or compute the value of a private field.
class Person { // Field private string name; // Property public string Name { get { return name; } set { name = value; } } }

13. Explain the concept of method overloading with an example.

Method overloading is the ability to define multiple methods with the same name but different signatures in the same class.

class MathOperations { public int Add(int a, int b) { return a + b; } public double Add(double a, double b) { return a + b; } } // Usage MathOperations math = new MathOperations(); int sum1 = math.Add(1, 2); // Outputs: 3 double sum2 = math.Add(1.5, 2.5); // Outputs: 4.0

14. What is method overriding in C#?

Method overriding is the ability of a subclass to provide a specific implementation of a method that is already defined in its base class.

class BaseClass { public virtual void Display() { Console.WriteLine("Base class display method"); } } class DerivedClass : BaseClass { public override void Display() { Console.WriteLine ("Derived class display method"); } }

15. How does the override keyword work in C#?

The override keyword is used to extend or modify a virtual/abstract method, property, indexer, or event in a base class into a derived class.

class BaseClass { public virtual void Display() { Console.WriteLine("Base class display method"); } } class DerivedClass : BaseClass { public override void Display() { Console.WriteLine("Derived class display method"); } }

16. What is the purpose of the abstract keyword in C#?

The abstract keyword is used to indicate that a class or a member is incomplete and must be implemented in a derived class. Abstract classes cannot be instantiated.

abstract class Animal { public abstract void MakeSound(); } class Dog : Animal { public override void MakeSound() { Console.WriteLine("Bark"); } }

17. What is a constructor and what is its purpose in C#?

A constructor is a special method in a class that is called when an instance of the class is created. Its purpose is to initialize the object.

class Person { public string Name { get; private set; } public Person(string name) { Name = name; } }

18. Can you explain what composition is in C#?

Composition is a design principle where a class is composed of one or more objects from other classes, implying a has-a relationship.

class Engine { public void Start() { Console.WriteLine("Engine started."); } } class Car { private Engine engine = new Engine(); public void Start() { engine.Start(); } }

19. What is the Dependency Inversion Principle?

The Dependency Inversion Principle is a design principle that suggests high-level modules should not depend on low-level modules. Both should depend on abstractions.

20. Can you explain dependency injection with an example?

Dependency injection is a technique where an object receives its dependencies from an external source rather than creating them itself.

interface IService { void Serve(); } class Service : IService { public void Serve() { Console.WriteLine("Service called."); } } class Client { private readonly IService _service; public Client(IService service) { _service = service; } public void Start() { _service.Serve(); } } // Usage IService service = new Service(); Client client = new Client(service); client.Start();

21. What is a task in terms of the Task Parallel Library (TPL)?

In the Task Parallel Library (TPL), a Task represents an asynchronous operation. It is a higher-level abstraction over threads and provides a way to execute code concurrently.

22. What are the differences between a delegate and an event in C#?

Delegates and events in C# are related but have key differences:

  • Delegate: A type that represents references to methods with a specific parameter list and return type.
  • Event: A way to provide notifications. Events are based on delegates and are used to publish/subscribe patterns.

23. What is the purpose of a constructor?

A constructor is a special method in a class that is called when an instance of the class is created. It is used to initialize objects.

24. Can we create an object of a static class?

No, static classes cannot be instantiated. They are used to create utility or helper classes that contain static members.
