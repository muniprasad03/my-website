Top 100 Entity Framework Interview Questions and Answers

1. What is Entity Framework?
Entity Framework (EF) is an open-source object-relational mapping (ORM) framework for .NET applications. It enables developers to work with databases using .NET objects, eliminating the need for most of the data-access code.
2. What are the advantages of using Entity Framework?
- Eliminates the need for most of the data-access code
- Supports LINQ queries
- Offers change tracking and lazy loading
- Reduces development time
3. What are the different approaches in Entity Framework?
- Database First
- Model First
- Code First
4. What is Code First approach?
In Code First, the database is created from the code. You define your model classes first, and then EF generates the database schema based on those classes.
5. What is Database First approach?
In Database First, you start with an existing database, and EF generates entity classes and the DbContext based on the database schema.
6. What is Model First approach?
In Model First, you design your data model visually using the EF Designer, and then EF generates both the database schema and the code from the model.
7. What is DbContext in Entity Framework?
DbContext is the primary class in EF that manages the database connection and is used to retrieve and save data. It represents a session with the database.
8. What is the difference between DbContext and ObjectContext?
DbContext is a wrapper around ObjectContext introduced in EF 4.1 to simplify coding. It provides a more streamlined API and is recommended for most modern applications. ObjectContext is more powerful but more complex and typically used in advanced scenarios.
9. What is a DbSet in Entity Framework?
A DbSet represents a collection of entities of a specific type that can be queried from the database and updated.
10. How does EF handle relationships?
EF handles relationships using navigation properties, foreign keys, and configurations through Data Annotations or Fluent API.
11. What is Lazy Loading?
Lazy Loading means related data is automatically loaded from the database when it is accessed through a navigation property.
12. What is Eager Loading?
Eager Loading is the process of loading related entities as part of the initial query using the Include method.
13. What is Explicit Loading?
Explicit Loading is when you manually load related entities using the Load method on a navigation property.
14. What are navigation properties?
Navigation properties in EF represent relationships between entities, like foreign keys in a relational database.
15. How can you configure relationships in EF?
Relationships can be configured using Data Annotations or Fluent API to specify multiplicity and foreign key constraints.
16. What is the Fluent API in Entity Framework?
Fluent API provides a way to configure entity classes using method chaining instead of attributes.
17. What is LINQ to Entities?
LINQ to Entities is a query language that allows you to write strongly typed queries against the Entity Framework context.
18. What is the difference between LINQ to Entities and LINQ to SQL?
LINQ to SQL works only with SQL Server and uses a different provider, while LINQ to Entities works with multiple databases via Entity Framework.
19. How do you update an entity in EF?
Retrieve the entity, make changes, and then call SaveChanges() on the DbContext.
20. What is the role of SaveChanges() in EF?
SaveChanges() commits all changes made in the context to the database.
21. How do you track changes in EF?
EF tracks changes using its Change Tracker, which monitors entity states such as Added, Modified, Deleted.
22. What is Change Tracking?
Change tracking is a feature where EF keeps track of changes made to entities so that the correct update statements are sent to the database.
23. How to disable tracking in EF?
Use AsNoTracking() for read-only queries to improve performance.
24. What is Migration in Entity Framework?
Migrations allow you to incrementally change the database schema over time as your model changes.
25. What command is used to add a migration?
Use the command: `Add-Migration MigrationName` in the Package Manager Console.
26. What is the use of Update-Database command?
It applies pending migrations to the database.
27. What is the difference between Add-Migration and Update-Database?
Add-Migration creates a migration file based on changes made to the model, while Update-Database applies migrations to the database.
28. How do you handle concurrency conflicts in Entity Framework?
Concurrency conflicts are handled by using optimistic concurrency, where the EF checks for conflicting changes before saving data. You can implement it using timestamps or version columns in the database.
29. What is a composite key in Entity Framework?
A composite key is a primary key made up of more than one column in a table. You can configure composite keys using the Fluent API.
30. How do you configure a one-to-many relationship in EF?
In EF, you can configure a one-to-many relationship by using navigation properties and foreign keys, either using Data Annotations or Fluent API.
31. What is a many-to-many relationship in Entity Framework?
A many-to-many relationship is a relationship where multiple records in one table are related to multiple records in another table. In EF, it is configured with a join table that stores the associations.
32. What is a stored procedure in EF, and how do you execute it?
A stored procedure is a precompiled SQL query that can be executed by the database. In EF, you can execute it using `DbContext.Database.SqlQuery` or `DbContext.Database.ExecuteSqlCommand`.
33. What is the difference between `First()` and `FirstOrDefault()`?
`First()` throws an exception if no element is found, whereas `FirstOrDefault()` returns `null` when no elements match the query.
34. What is the `AsNoTracking()` method used for?
`AsNoTracking()` is used to query data without tracking changes in the context, which improves performance for read-only queries.
35. How do you perform batch updates in Entity Framework?
EF does not provide built-in batch updates, but you can update entities in bulk by modifying them in memory and then calling `SaveChanges()` for all entities.
36. What is Lazy Loading in EF?
Lazy Loading automatically loads related entities when they are accessed for the first time. It is enabled by default in EF.
37. How do you prevent Lazy Loading in EF?
You can disable Lazy Loading by setting `LazyLoadingEnabled` to `false` in the `DbContext` configuration or by using `AsNoTracking()` for read-only queries.
