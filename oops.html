C# OOP Interview Questions and Answers

1. What is Object-Oriented Programming (OOP)?

Object-Oriented Programming (OOP) is a programming paradigm based on the concept of "objects," which can contain data in the form of fields and code in the form of procedures. OOP focuses on the organization of software design around data, or objects, rather than functions and logic. It aims to increase the modularity and reusability of code.

2. What are the main principles of OOP?

The main principles of OOP are:
  • Encapsulation: Bundling the data with the methods that operate on that data.
  • Abstraction: Hiding complex implementation details and showing only the necessary features of an object.
  • Inheritance: Creating a new class based on an existing class, inheriting its members and behaviors.
  • Polymorphism: Allowing objects to be treated as instances of their parent class rather than their actual class, enabling methods to do different things based on the object it is acting upon.

3. What is encapsulation in C#?

Encapsulation is the practice of hiding the internal state and requiring all interaction to be performed through an object's methods. It is achieved by using access modifiers to restrict direct access to some of an object's components.
public class Person
{
    // Private field
    private string name;

    // Public property
    public string Name
    {
        get { return name; }
        set { name = value; }
    }

    // Method to display the name
    public void Display()
    {
        Console.WriteLine($"Name: {Name}");
    }
}

4. How is encapsulation implemented in C#?

Encapsulation in C# is implemented using access modifiers. By using keywords like `private`, `protected`, and `public`, you can control the visibility and accessibility of class members. Here is an example:
public class BankAccount
{
    // Private field
    private decimal balance;

    // Public method to deposit money
    public void Deposit(decimal amount)
    {
        if (amount > 0)
        {
            balance += amount;
        }
    }

    // Public method to get the balance
    public decimal GetBalance()
    {
        return balance;
    }
}

5. What is abstraction in C#?

Abstraction is the concept of hiding the complex implementation details and showing only the essential features of an object. In C#, abstraction is achieved using abstract classes and interfaces.

6. How do you create an abstract class in C#?

An abstract class is a class that cannot be instantiated on its own and must be inherited by other classes. It can contain abstract methods (methods without implementation) that must be implemented by derived classes. Here is an example:
public abstract class Shape
{
    public abstract double GetArea();
}

public class Rectangle : Shape
{
    public double Width { get; set; }
    public double Height { get; set; }

    public override double GetArea()
    {
        return Width * Height;
    }
}

7. What is the difference between an abstract class and an interface in C#?

An abstract class can provide default behavior and state, while an interface can only define method signatures and properties without implementation. A class can implement multiple interfaces but can inherit only one abstract class. Abstract classes are used for providing common base functionality, whereas interfaces are used for defining contracts that multiple classes can adhere to.

8. How do you implement inheritance in C#?

Inheritance is implemented in C# using the `:` symbol. The derived class inherits from the base class and can override or add new members. Here is an example:
public class Animal
{
    public void Eat()
    {
        Console.WriteLine("Eating...");
    }
}

public class Dog : Animal
{
    public void Bark()
    {
        Console.WriteLine("Woof!");
    }
}

public class Program
{
    public static void Main()
    {
        Dog myDog = new Dog();
        myDog.Eat(); // Inherited method
        myDog.Bark(); // Derived class method
    }
}

9. What is polymorphism in C#?

Polymorphism allows objects to be treated as instances of their base class rather than their actual class. It enables methods to have different behaviors based on the object that is calling them. There are two types of polymorphism in C#: compile-time (method overloading) and runtime (method overriding).

10. How do you achieve polymorphism through method overriding in C#?

Polymorphism through method overriding is achieved using the `virtual` keyword in the base class and the `override` keyword in the derived class. Here is an example:
public class Animal
{
    public virtual void MakeSound()
    {
        Console.WriteLine("Some sound...");
    }
}

public class Dog : Animal
{
    public override void MakeSound()
    {
        Console.WriteLine("Woof!");
    }
}

public class Program
{
    public static void Main()
    {
        Animal myAnimal = new Dog();
        myAnimal.MakeSound(); // Outputs: Woof!
    }
}

11. What is an interface in C#?

An interface in C# is a contract that defines a set of methods and properties without providing implementation. Classes and structs that implement the interface must provide implementations for all of its members. Interfaces are used to achieve abstraction and multiple inheritance.

12. How do you define and implement an interface in C#?

An interface is defined using the `interface` keyword. Classes or structs that implement the interface use the `:` symbol and must provide concrete implementations for the interface's members. Here is an example:
public interface IAnimal
{
    void Eat();
    void MakeSound();
}

public class Dog : IAnimal
{
    public void Eat()
    {
        Console.WriteLine("Eating...");
    }

    public void MakeSound()
    {
        Console.WriteLine("Woof!");
    }
}

public class Program
{
    public static void Main()
    {
        IAnimal myDog = new Dog();
        myDog.Eat();
        myDog.MakeSound();
    }
}

13. Can an interface have properties in C#?

Yes, an interface can have properties. The properties defined in an interface only include the property signature (i.e., the name, type, and accessors) but do not include implementation. Classes that implement the interface must provide an implementation for these properties.
public interface IShape
{
    double Area { get; }
}

public class Circle : IShape
{
    public double Radius { get; set; }

    public double Area
    {
        get { return Math.PI * Radius * Radius; }
    }
}

14. What is a virtual method in C#?

A virtual method in C# is a method defined in a base class that can be overridden in derived classes. The base class provides a default implementation, but derived classes can provide a specific implementation using the `override` keyword.
public class BaseClass
{
    public virtual void Display()
    {
        Console.WriteLine("Base class display method.");
    }
}

public class DerivedClass : BaseClass
{
    public override void Display()
    {
        Console.WriteLine("Derived class display method.");
    }
}

15. What is method overloading in C#?

Method overloading is a feature in C# that allows a class to have more than one method with the same name, but with different parameters (different type or number of parameters). It is a form of compile-time polymorphism.
public class Calculator
{
    public int Add(int a, int b)
    {
        return a + b;
    }

    public double Add(double a, double b)
    {
        return a + b;
    }

    public int Add(int a, int b, int c)
    {
        return a + b + c;
    }
}

16. What is method overriding in C#?

Method overriding is a feature in C# that allows a derived class to provide a specific implementation of a method that is already defined in its base class. This is achieved using the `override` keyword in the derived class and the `virtual` keyword in the base class.
public class Animal
{
    public virtual void MakeSound()
    {
        Console.WriteLine("Animal sound");
    }
}

public class Cat : Animal
{
    public override void MakeSound()
    {
        Console.WriteLine("Meow");
    }
}

17. What is constructor overloading in C#?

Constructor overloading in C# is a concept where multiple constructors can be defined within a class, each with a different parameter list. This allows objects of the class to be initialized in different ways.
public class Person
{
    public string Name { get; set; }
    public int Age { get; set; }

    public Person()
    {
    }

    public Person(string name)
    {
        Name = name;
    }

    public Person(string name, int age)
    {
        Name = name;
        Age = age;
    }
}

18. What is a static class in C#?

A static class in C# is a class that cannot be instantiated and can only contain static members. It is used to group related methods and properties that do not require an instance of the class. Static classes are commonly used for utility or helper methods.
public static class MathUtils
{
    public static int Add(int a, int b)
    {
        return a + b;
    }

    public static double Multiply(double a, double b)
    {
        return a * b;

19. What is the difference between a class and a struct in C#?

In C#, classes are reference types and are allocated on the heap, while structs are value types and are allocated on the stack. Classes support inheritance, whereas structs do not. Classes can be instantiated with or without the `new` keyword, whereas structs are always instantiated with `new`.
public class MyClass

    public int Value { get; set; }

public struct MyStruct

    public int Value { get; set; }

22. Can you inherit from multiple classes in C#?

No, C# does not support multiple inheritance of classes. A class can inherit from only one base class. However, a class can implement multiple interfaces, which allows it to achieve a form of multiple inheritance.

23. What is the use of the `base` keyword in C#?

The `base` keyword in C# is used to access members of the base class from within a derived class. It can be used to call base class methods, constructors, and properties.
public class Animal
{
    public void Eat()
    {
        Console.WriteLine("Animal eating...");
    }
}

public class Dog : Animal
{
    public void Display()
    {
        base.Eat(); // Calls the Eat method from Animal class
    }
}

24. What is method hiding in C#?

Method hiding occurs when a derived class defines a method with the same name as a method in the base class, but without using the `override` keyword. This hides the base class method. Method hiding is achieved using the `new` keyword.
public class BaseClass
{
    public void Display()
    {
        Console.WriteLine("Base class display");
    }
}

public class DerivedClass : BaseClass
{
    public new void Display()
    {
        Console.WriteLine("Derived class display");
    }
}

25. What is the purpose of the `sealed` keyword in C#?

public sealed class FinalClass

26. How do you prevent a method from being overridden in C#?

To prevent a method from being overridden, you can mark it with the `sealed` keyword in a derived class. The method must be marked as `override` in the derived class to be sealed. The base class method should be marked as `virtual`.
public class BaseClass

    {
        Console.WriteLine("Base class display");
    }

public class DerivedClass : BaseClass

public class MyCollection

28. What is the difference between `public`, `private`, `protected`, and `internal` access modifiers?

The `public` access modifier allows access from any code. The `private` modifier restricts access to the containing class. The `protected` modifier allows access from derived classes. The `internal` modifier restricts access to the same assembly. The `protected internal` modifier allows access from derived classes and the same assembly.

29. What is the `new` keyword used for in C#?

The `new` keyword in C# is used to create instances of objects and to hide base class members. It can be used to instantiate a class or to create a new member in a derived class that hides a member in the base class.

31. What is a destructor in C#?

public class MyClass

32. What is the `static` keyword used for in C#?

The `static` keyword is used to declare members that belong to the type itself rather than to a specific instance. Static members can be accessed directly through the class name and do not require an instance of the class.

33. What is the difference between `const` and `readonly` in C#?

`const` is used to declare compile-time constants, while `readonly` is used to declare runtime constants that can only be assigned in the constructor. `const` values must be initialized at the time of declaration, whereas `readonly` values can be assigned in the constructor or during declaration.

34. What is the purpose of the `this` keyword in C#?

The `this` keyword refers to the current instance of the class. It is used to access members (fields, methods, and properties) of the current object, and to distinguish between instance members and parameters with the same name.
public class Car
{
    private string model;

    public Car(string model)
    {
        this.model = model; // Uses 'this' to refer to the instance field
    }
}

35. What is the difference between `override` and `new` in C#?

The `override` keyword is used to provide a new implementation of a virtual method inherited from a base class. The `new` keyword is used to hide a base class member with a new member in the derived class. `override` maintains polymorphic behavior, while `new` does not.

36. How does exception handling work in C#?

Exception handling in C# is achieved using the `try`, `catch`, `finally`, and `throw` keywords. Code that might throw an exception is placed inside a `try` block, exceptions are caught in `catch` blocks, and any cleanup code is placed in the `finally` block. Exceptions can be thrown using the `throw` keyword.
try
{
    // Code that may throw an exception
}
catch (Exception ex)
{
    // Code to handle the exception
}
finally
{
    // Code that always executes, regardless of an exception
}

37. What is the purpose of the `using` statement in C#?

The `using` statement is used to ensure that resources are properly disposed of after use. It is commonly used with objects that implement the `IDisposable` interface. When the `using` block is exited, the `Dispose` method is automatically called to release resources.
using (var stream = new FileStream("file.txt", FileMode.Open))
{
    // Use the stream
}
// The stream is automatically disposed of here

38. What is the difference between `IEnumerable` and `IQueryable` in C#?

`IEnumerable` is used for querying collections in memory and is executed in-memory. `IQueryable` is used for querying data sources that support LINQ (e.g., databases) and is executed in the database or data source. `IQueryable` allows for more efficient queries as it translates queries into the native query language of the data source.

39. What is a lambda expression in C#?

A lambda expression is a concise way to define anonymous methods using the `=>` syntax. It is often used with LINQ queries and delegates. Lambda expressions can have zero or more parameters and a single expression or statement block.
Func square = x => x * x;

40. What is the purpose of the `params` keyword in C#?

PrintNumbers(1, 2, 3, 4, 5); // Prints numbers 1 through 5

41. What is the difference between `ref` and `out` parameters in C#?

Both `ref` and `out` parameters are used to pass arguments by reference. The `ref` parameter requires that the variable be initialized before being passed, while the `out` parameter does not. `out` parameters must be assigned a value before the method returns.

44. What is garbage collection in C#?

Garbage collection is an automatic memory management feature in C# that reclaims memory occupied by objects that are no longer in use. The garbage collector (GC) periodically checks for objects that are no longer referenced and releases their memory to avoid memory leaks.

45. What is the difference between `object` and `dynamic` in C#?

The `object` type is the base type of all other types in C# and is statically typed, meaning its type is checked at compile time. The `dynamic` type bypasses compile-time type checking and is resolved at runtime, allowing for more flexibility but with potential for runtime errors.

46. What is a `delegate` in C#?

A delegate is a type that represents references to methods with a specific parameter list and return type. Delegates are used to pass methods as arguments and can be used to implement event handling and callback methods.
public delegate void MyDelegate(string message);

public class Program
{
    public static void DisplayMessage(string message)
    {
        Console.WriteLine(message);
    }

    public static void Main()
    {
        MyDelegate del = DisplayMessage;
        del("Hello, world!");
    }
}

47. What are events in C#?

Events in C# are a way to provide notifications when something of interest occurs. They are based on delegates and allow objects to notify other objects when certain actions take place. Events are declared using the `event` keyword.
public class Button
{
    public event EventHandler Click;

    protected virtual void OnClick()
    {
        Click?.Invoke(this, EventArgs.Empty);
    }
}

The `async` keyword is used to declare a method as asynchronous, allowing it to use the `await` keyword. The `await` keyword is used within an `async` method to wait for the completion of a task without blocking the executing thread, facilitating asynchronous programming and improving responsiveness.
public async Task GetDataAsync()

49. What is LINQ in C#?

LINQ (Language Integrated Query) is a set of features in C# that allows for querying and manipulating data using a syntax integrated into the language. LINQ can be used with various data sources like collections, databases, XML, etc., using query syntax or method syntax.
var numbers = new List { 1, 2, 3, 4, 5 };
var evenNumbers = from number in numbers
                   where number % 2 == 0
                   select number;

50. What is the difference between `String` and `StringBuilder` in C#?

`String` is immutable, meaning any modification results in a new string instance being created. `StringBuilder` is mutable and allows for efficient string manipulation by modifying the string in place. `StringBuilder` is preferable when performing many modifications to a string.
StringBuilder sb = new StringBuilder();
sb.Append("Hello");
sb.Append(" World");
string result = sb.ToString(); // "Hello World"
