Interview Questions and Answers

1. Do you know Multithreading and have you used multithreading in your Projects?

Multithreading allows a program to run multiple threads concurrently, improving performance by doing multiple tasks at once. Examples include performing background tasks like file processing, web requests, or UI updates in desktop applications to keep the UI responsive.

2. When you create the following, where does the memory allocation happen.

Class: Memory allocation for class methods and static members occurs in the method area.
Object: Memory for objects is allocated on the heap.

3. Can you Explain what is Object Oriented Programming with Advantages?

Object-Oriented Programming (OOP) is a programming paradigm based on the concept of "objects" that contain data and methods. Advantages include code reusability, scalability, ease of maintenance, and abstraction.

4. When you store an Object on heap, how is the size of the object calculated.

The size of an object includes the size of its instance variables plus any overhead for object management by the runtime.

5. When you add a class member in a class, will the size of the object increase.

Yes, adding a class member will increase the size of an object because the new member requires additional memory.

6. Different types of Constructors in C#

Default constructor, Parameterized constructor, Copy constructor, Static constructor, Private constructor.

7. Can we have static members in non-static class?

Yes, static members can be defined in a non-static class. They belong to the class rather than any instance.

8. Consider a class Account, which has class members like int bank balance, string AccountholderName. Where does the memory allocation of these class members take place?

Non-static class members (like bankBalance and AccountholderName) are allocated on the heap.

9. When memory allocation will happen for static members and non-static members

Static members: Allocated when the class is loaded.
Non-static members: Allocated when an instance of the class is created.

10. Can we restrict a class from creating objects?

Yes, use a private constructor to prevent instantiation. This is often used in Singleton patterns.

11. Can we call a Constructor explicitly?

Constructors cannot be called explicitly like methods. They are called when an object is instantiated.

12. Have you used delegates in C#? What are delegates? Why do we use it? Explain with advantages.

Delegates are type-safe function pointers used to encapsulate method references. They are used for callback methods, implementing event handling, and designing extensible and flexible applications. Advantages include flexibility, type safety, and encapsulation of method calls.

13. Have you created any custom events in your projects?

Custom events are created using delegates. They enable classes to notify other classes or objects when something of interest occurs.

14. Different Types of classes in C#

Abstract class, Sealed class, Static class, Partial class, Nested class.

15. If you have used sealed class and Static class in your project, why did you use it. Explain the situation.

Sealed class: Used to prevent inheritance, often for security or design reasons.
Static class: Used to group related static members, without needing to instantiate the class.

16. Disadvantages of using static class?

Cannot be instantiated, limited to static members only, and cannot implement interfaces.

17. Difference between Encapsulation and abstraction? If possible, explain with real-time Example.

Encapsulation: Hiding the internal state and requiring all interaction to be performed through an object's methods.
Abstraction: Simplifying complex systems by modeling classes appropriate to the problem, using abstract classes or interfaces.
Example: A car (encapsulation hides the engine's complexity; abstraction uses the concept of a Vehicle).

18. In my Model, I have a background task running. Once the background task is completed UI should be notified and be updated. How do we achieve it?

Use async programming with tasks and callbacks, or event-driven mechanisms to notify the UI thread for updates.

19. There are two classes. Class Person contains Name, age, DOB. Class Employee contains DOJ, etc. Can you explain the relationship between the two classes?

Employee can inherit from Person (inheritance), or have a Person as a member (composition).

20. Purpose of encapsulation?

Encapsulation protects the object's integrity by preventing external code from making uncontrolled changes.

21. Serialization?

Serialization is the process of converting an object into a format that can be stored or transmitted. Formats include XML, JSON, Binary. Usage in WCF enables easy data transmission between client and server.

22. Inheritance vs. Composition?

Inheritance is an "is-a" relationship, composition is a "has-a" relationship. Composition is often preferred for its flexibility and reducing tight coupling.

23. What is unit testing?

Unit testing is the testing of individual units or components of a software. It helps ensure that each part of the code works correctly.

24. What is composition?

Composition is a design principle in object-oriented programming where one class contains an instance of another class. This allows complex types to be built by combining objects of other types, promoting code reuse and flexibility.

25. Dependency Inversion Principle (DIP)? Explain with constructor.

The Dependency Inversion Principle states that high-level modules should not depend on low-level modules. Both should depend on abstractions. Also, abstractions should not depend on details. Details should depend on abstractions.

26. Class A is dependent on class B and an object to call class B – how do you remove dependency from one class to another class – avoid static class?

To remove or minimize dependency between classes, you can use Dependency Injection (DI). This involves passing dependencies into a class rather than creating them within the class.

27. Inversion of Control (IoC) Principles?

Inversion of Control (IoC) refers to a design principle where the control of objects or portions of a program is transferred to a container or framework. It often uses dependency injection, service locators, or factory patterns.

28. What is a task in terms of TPL?

In the Task Parallel Library (TPL), a Task represents an asynchronous operation. It is a higher-level abstraction over threads and provides a way to execute code concurrently.

29. Delegate and event differences?

Delegates and Events in C# are related but have key differences:

  • Delegate: A type that represents references to methods with a specific parameter list and return type.
  • Event: A way to provide notifications. Events are based on delegates and are used to publish/subscribe patterns.

30. What is the use of a constructor?

A constructor is a special method in a class that is called when an instance of the class is created. It is used to initialize objects.

31. Static class – can we create object?

Static classes cannot be instantiated. They are used to create utility or helper classes that contain static members.
