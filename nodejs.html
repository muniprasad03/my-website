Prepare for your next Node.js interview with the top 200 questions and answers!

1. What is Node.js?

Answer: Node.js is an open-source, cross-platform JavaScript runtime environment that executes JavaScript code outside of a browser. It is based on the V8 JavaScript engine, which is used in Google Chrome.

2. What are the advantages of Node.js?

Answer: Some of the main advantages of Node.js include:

  • Fast and efficient due to V8 engine.
  • Asynchronous and event-driven architecture.
  • Single-threaded with non-blocking I/O operations.
  • Scalable for building web applications.
  • Large number of modules available in npm.

3. What is the event loop in Node.js?

Answer: The event loop is a fundamental concept in Node.js that enables non-blocking, asynchronous operations. Node.js uses the event loop to handle I/O operations, making it highly scalable and efficient.

4. What is the difference between `process.nextTick()` and `setImmediate()`?

Answer: Both `process.nextTick()` and `setImmediate()` schedule a callback to be executed in the next iteration of the event loop, but:

  • `process.nextTick()` will execute the callback after the current operation completes but before any event loop operations.
  • `setImmediate()` will execute the callback after the current event loop phase has completed.

5. What is the use of `require()` in Node.js?

Answer: `require()` is a built-in function in Node.js used to import modules, JSON files, and local files into your application. It enables the use of external libraries and APIs.

6. How do you create a basic HTTP server in Node.js?

Answer: You can create a basic HTTP server using the built-in `http` module. Example:

const http = require('http');

http.createServer((req, res) => {
    res.writeHead(200, { 'Content-Type': 'text/plain' });
    res.end('Hello, World!');
}).listen(8080, () => {
    console.log('Server is running at http://localhost:8080/');
});

121. What is the `process.nextTick()` method in Node.js?

157. How does the event loop work in Node.js?

Answer: The event loop in Node.js is the mechanism that allows non-blocking I/O operations. It constantly checks the call stack and the event queue.
