Top 100 .NET Core Interview Questions and Answers

1. What is .NET Core?
.NET Core is an open-source, cross-platform framework for building modern, cloud-based, and internet-connected applications. It is a runtime for running applications developed with .NET technologies, and it is the foundation for .NET 5 and later versions.
2. What are the main differences between .NET Core and .NET Framework?
.NET Core is cross-platform, meaning it can run on Windows, macOS, and Linux. .NET Framework is limited to Windows. .NET Core is modular and can be updated more frequently, while .NET Framework is more monolithic and has a slower update cycle.
3. What is the purpose of the `Startup` class in .NET Core?
The `Startup` class is used to configure the services and the application's request pipeline. It contains methods such as `ConfigureServices` to add services to the DI container and `Configure` to define the request handling pipeline.
4. What is Dependency Injection (DI) and how is it used in .NET Core?
Dependency Injection is a design pattern used to implement Inversion of Control (IoC), where the container is responsible for providing the dependencies of a class. In .NET Core, DI is built into the framework and is configured in the `Startup` class.
5. What is Middleware in .NET Core?
Middleware is software that is assembled into an application pipeline to handle requests and responses. Each component in the pipeline can perform operations before and after the next component in the pipeline. Middleware components are added in the `Configure` method of the `Startup` class.
6. Explain the concept of Entity Framework Core.
Entity Framework Core (EF Core) is an Object-Relational Mapper (ORM) for .NET Core that allows developers to interact with a database using .NET objects. It supports LINQ queries, change tracking, and database migrations.
7. What are some common ways to handle configuration in .NET Core?
Configuration in .NET Core is typically handled using the `IConfiguration` interface, which can read configuration settings from various sources, including JSON files, environment variables, and command-line arguments. The `appsettings.json` file is commonly used for configuration.
8. What is Kestrel?
Kestrel is the cross-platform web server used by default in .NET Core applications. It is a lightweight, high-performance server that is optimized for serving ASP.NET Core applications.
9. How do you create a web API in .NET Core?
To create a web API in .NET Core, you can use the ASP.NET Core Web API template. Define controllers that inherit from `ControllerBase`, and use attributes like `[HttpGet]`, `[HttpPost]`, etc., to define API endpoints. The `Startup` class configures routing and middleware for the API.
10. What is the role of the `appsettings.json` file?
The `appsettings.json` file is used to store configuration settings for an application. It is commonly used to define settings such as connection strings, application-specific settings, and environment-specific configurations.
11. How do you implement logging in .NET Core?
Logging in .NET Core is implemented using the built-in `ILogger` interface, which can be injected into services and controllers. The logging framework supports various providers, such as console, debug, and file, and can be configured in the `Startup` class.
12. What is the difference between `IServiceCollection` and `IServiceProvider`?
`IServiceCollection` is used to register services and configure the dependency injection container, while `IServiceProvider` is used to resolve services from the container. `IServiceProvider` provides an interface for obtaining service instances.
13. Explain the purpose of the `Configure` method in the `Startup` class.
The `Configure` method in the `Startup` class is used to define the middleware pipeline for handling HTTP requests and responses. It sets up middleware components that process requests, handle routing, and generate responses.
14. What are Razor Pages?
Razor Pages is a page-based programming model in ASP.NET Core for building web applications. It is a simpler alternative to MVC for scenarios where the application consists primarily of pages rather than a complex set of controllers and views.
15. How does .NET Core handle cross-origin requests?
Cross-Origin Resource Sharing (CORS) in .NET Core is handled using the `Microsoft.AspNetCore.Cors` middleware. CORS policies are defined in the `Startup` class and control which origins, headers, and methods are allowed for cross-origin requests.

In ASP.NET Core, filters are used to execute code before or after an action method runs. The types of filters include Authorization filters, Resource filters, Action filters, Exception filters, and Result filters.

22. Explain the concept of `Service Lifetime` in .NET Core.
In .NET Core, service lifetime determines how long a service instance is kept alive. The three main lifetimes are:
  • Transient: A new instance is created each time it is requested.
  • Scoped: A new instance is created once per request (or per scope).
  • Singleton: A single instance is created and shared throughout the application's lifetime.
