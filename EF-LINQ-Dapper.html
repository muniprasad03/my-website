Entity Framework

1. What is Entity Framework?
Entity Framework (EF) is an open-source object-relational mapper (ORM) framework for .NET applications. It allows developers to work with a database using .NET objects, eliminating the need for most of the data-access code.
2. What is the difference between Entity Framework and ADO.NET?
Entity Framework is an ORM that provides a higher-level abstraction for database access compared to ADO.NET, which is a lower-level data access technology. EF handles object-relational mapping, change tracking, and more, while ADO.NET focuses on raw data operations.
50. What is lazy loading in Entity Framework?
Lazy loading is a technique where related data is not loaded until it is specifically requested. In Entity Framework, lazy loading is enabled by default for navigation properties if you use virtual navigation properties and proxies.

LINQ

51. What is LINQ?
Language Integrated Query (LINQ) is a set of methods and operators in .NET that allows you to query collections of data in a consistent manner. It provides a query syntax that integrates with C# and VB.NET, allowing you to write queries directly in your code.
52. What are the different types of LINQ queries?
LINQ queries can be categorized into three types: LINQ to Objects (queries on in-memory collections), LINQ to SQL (queries on SQL databases), and LINQ to Entities (queries on Entity Framework entities).
100. How does LINQ handle deferred execution?
LINQ supports deferred execution, meaning that the query is not executed until its results are iterated over. This allows for building up complex queries and executing them efficiently, but also means that the data can change between query construction and execution.

Dapper

101. What is Dapper?
Dapper is a lightweight, high-performance micro ORM for .NET. It is used to map query results to objects and execute SQL queries directly, providing a balance between raw ADO.NET and full ORM frameworks like Entity Framework.
102. How does Dapper differ from Entity Framework?
Dapper is designed to be simpler and faster than Entity Framework. It does not provide all the features of a full ORM like EF, such as change tracking and complex relationship management, but offers better performance and simplicity for basic data access needs.
150. How do you handle transactions in Dapper?
Transactions in Dapper are handled using the `IDbTransaction` interface from ADO.NET. You start a transaction by calling `BeginTransaction` on an `IDbConnection` object, and then pass the transaction object to Dapper's query methods.
